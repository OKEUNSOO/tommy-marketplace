# Storybank Format

STAR 답변을 누적 저장하는 스토리뱅크의 구조와 관리 기준.

## 테이블 구조

| 필드 | 설명 |
|------|------|
| **ID** | 고유 식별자 (S001, S002, ...) |
| **제목** | 구체적이고 기억하기 쉬운 이름 (예: "전사 지표 정의를 혼자 뒤집은 경험") |
| **핵심 역량** | 이 스토리가 증명하는 주요 역량 |
| **보조 역량** | 추가로 드러나는 역량 (선택) |
| **임팩트** | 수치화된 결과, 불가능하면 정성적 설명 |
| **도메인** | 기술 / 제품 / 비즈니스 / 사람 |
| **리스크/이해관계** | 잘못됐으면 무슨 일이 생겼을까? |
| **Earned Secret** | 이 경험에서만 얻을 수 있었던 역발상 인사이트 |
| **강점** | 1~5점 (5: 차별화, 수치화, 기억에 남음 / 3: 탄탄하지만 평범 / 1: 거의 못 씀) |
| **사용 횟수** | 실제 면접에서 사용한 총 횟수 |
| **마지막 사용** | 가장 최근 사용일 |
| **메모** | 피드백, 반응, 개선 포인트 |

## Earned Secret 개념

Earned Secret은 그 경험을 직접 겪은 사람만 알 수 있는 역발상 인사이트다.
단순한 교훈("팀워크가 중요하다")이 아니라, 직접 경험하지 않으면 모르는 구체적 발견이어야 한다.

추출 질문:
- "이 경험에서 나만 알게 된 것은 무엇인가?"
- "비슷한 스토리를 가진 다른 사람과 내 스토리가 다른 점은?"
- "예상과 반대로 작동했던 부분이 있었나?"

## 스토리뱅크 건강 기준

**건강한 스토리뱅크:**
- 8~12개 스토리 인덱싱
- 강점 4+ 최소 5개
- 핵심 역량 전체 커버 (기술, 협업, 문제해결, 리더십 등)
- 도메인 믹스 (한 종류에 치우치지 않음)
- 수치화된 결과 최소 2개
- 실패/학습 스토리 최소 1개
- 같은 스토리 3회 이상 사용 금지

**경고 신호:**
- 스토리 6개 미만
- 대부분 강점 3 이하
- 같은 회사/시기 스토리만 있음
- 실패 스토리 없음
- 2~3개 스토리에만 의존

## 저장 위치

`career-notes/01_경험/storybank.md`

## 저장 형식

```markdown
## [S001] 제목 — YYYY-MM-DD

**핵심 역량**: ...
**임팩트**: ...
**Earned Secret**: ...
**강점**: /5

**S (상황)**: ...
**T (과제)**: ...
**A (행동)**: ...
**R (결과)**: ...
```
