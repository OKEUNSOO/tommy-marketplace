# Application Workflow

Use this when the user asks for JD matching, resume tailoring, 자기소개서, 면접 answers, or application strategy.

## JD Matching

1. Extract JD keywords:
   - Role mission
   - Required skills
   - Preferred skills
   - Domain/business model
   - Expected outcomes or KPIs
2. Match against:
   - `05_Domain_Tag_Layer.md`
   - `02_Skill_Layer.md`
   - `01_Project_Layer.md`
   - `03_Metric_Layer.md`
3. Return:
   - Strong matches
   - Partial matches
   - Missing or weak evidence
   - Recommended project priority

## Resume Tailoring

Produce bullets in this order:

1. Most JD-relevant project
2. Strongest quantitative outcome
3. Supporting technical or analytical capability
4. Optional portfolio/tooling project

Avoid bullets that only list tools. Every bullet needs a problem and an outcome.

## 자기소개서

Use this structure:

1. Company/JD task: mention a specific problem or business need.
2. Matching experience: connect one or two projects from source career notes.
3. Contribution: explain how the user would apply the same pattern.
4. Growth: mention a concrete area to deepen, not generic passion.

Avoid:

- "비전에 공감했습니다" without evidence.
- "배우고 싶습니다" as the main reason.
- Overclaiming direct industry expertise when only adjacent evidence exists.

## Interview Answers

Use STAR internally:

- Situation: what was unclear or broken.
- Task: what the user needed to decide or build.
- Action: analysis, implementation, tradeoffs.
- Result: metric, decision impact, award, or learning.

Final answer should sound spoken, not like a report.

## Company Research Frame

When company information is provided or discoverable from local docs, structure it as:

- What the company sells or operates.
- Likely core KPI.
- Why this role matters.
- Where the user's experience connects.
- Smart questions to ask in interview.
