# Career Workspace Folder Map

Workspace root:

```text
career-notes/
```

Career evidence root is the user's current evidence workspace. Recommended layout:

```text
career-notes/
├── 01_경험/
├── 02_서류/
└── 03_지원/
```

## Core Folders

| Path | Purpose |
| --- | --- |
| `01_경험/` | Source of truth for project, internship, education, and analysis evidence. |
| `02_서류/` | Resume, portfolio, 자기소개서 source material. |
| `03_지원/` | Company-specific application packages and JD notes. |
| `skills/` | Project-local Codex skills and deterministic helper scripts. |

## Files To Read First

1. `README.md`
2. `01_경험/경험정리_인덱스.md`, when present
3. `02_서류/이력서.md`, when present
4. `02_서류/포트폴리오.md`, when present
5. `skills/career-workspace/references/skill-map.md`
6. `skills/company-research/references/research-frame.md`

## Project Positioning

- Career evidence notes: canonical user evidence.
- `skills/career-experience-db`: JD-matched context from the vault.
- `skills/job-post-markdown`: job posting URL to Markdown.
- `skills/company-research`: company research and application strategy playbook.
