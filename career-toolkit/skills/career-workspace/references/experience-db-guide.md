# Career Evidence Guide

Use the user's local career evidence notes as the canonical evidence database.

## Read Order

1. `01_경험/경험정리_인덱스.md`: experience index and navigation, when present.
2. `01_경험/`: project, internship, education, and analysis notes.
3. `02_서류/이력서.md`: current resume positioning, when present.
4. `02_서류/포트폴리오.md`: project portfolio wording, when present.
5. `03_지원/`: company-specific JD, resume, portfolio, and interview material, when present.

## Evidence Shape

When auditing or adding an experience note, make sure the usable evidence is recoverable:

- Project or role name and period.
- Domain: SaaS, e-commerce, public policy, recommendation, marketing, EdTech, etc.
- Problem type: funnel, retention, segmentation, KPI design, prediction, recommendation, geospatial analysis.
- Data type: logs, location data, text, ad performance, transactions, learning data.
- Tools and methods.
- Key metrics and quantitative outcomes.
- Decision impact.
- Limitations and next improvement.

## Reuse Rules

- Resume bullets should use action + method + result.
- 자기소개서 and interview answers should connect problem, action, result, and learning.
- Do not invent projects, periods, metrics, tools, or contribution levels.
- If proof is missing, write `보강 필요` and name the missing field.
- When evidence changes, update the original source note first.
