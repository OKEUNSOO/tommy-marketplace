텍스트의 문법·논리·흐름 오류를 찾아서 핀포인트로 수정안을 제시해줘. 전체를 다시 쓰지 말고 꼭 필요한 부분만 고쳐줘.

입력: $ARGUMENTS

## 작업 순서

### 1단계: 텍스트 수집
붙여넣기, 파일 경로(DOCX, PDF, 마크다운), 이메일 초안 등 어떤 형태든 받아.

### 2단계: 세 가지 카테고리로 분석

**문법**: 맞춤법, 띄어쓰기, 주어-서술어 호응, 시제 일관성, 조사 오류, 문장 완결성
**논리**: 모순, 근거 없는 주장, 순환 논리, 불명확한 지시어, 비약
**흐름**: 어색한 연결, 문장 리듬, 단락 구조, 중복 표현, 가독성

### 3단계: 교정 보고서 출력

```
## 교정 보고서

**텍스트 길이**: [글자 수/단어 수]
**발견된 문제**: [카테고리별 건수]

### 문제 목록

#### 1. [카테고리: 문법/논리/흐름]
- **위치**: "[문제 있는 원문 인용]"
- **문제**: [무엇이 잘못됐는지]
- **수정안**: "[수정된 텍스트]"

[각 문제마다 반복]

### 요약
- 문법: [X]건
- 논리: [X]건
- 흐름: [X]건
- 전반적 품질: [평가]
```

### 4단계: 제안
- "모든 수정을 반영해서 교정본을 드릴까요?"
- "특정 섹션을 더 자세히 볼까요?"

## 주의사항

- 수정은 최소화 — 꼭 필요한 것만 바꾸고, 필자의 문체를 살린다
- 스타일 차이와 실제 오류를 구분한다 — 다를 뿐이지 틀린 게 아닌 경우도 있음
- 자소서·이력서 등 전문 문서는 어조 일관성과 독자 적합성도 확인
- 한국어 취업 서류의 경우: 존댓말/개조식 혼용 여부, 불필요한 수식어 제거도 체크
