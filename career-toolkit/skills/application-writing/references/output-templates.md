# Application Output Templates

## Resume Bullet

```text
- [역할/행동] [분석/구현 방법]을 통해 [문제]를 해결하고, [정량/정성 결과]를 만들었습니다.
```

Checklist:

- Use an action verb.
- Include method or tool when it strengthens fit.
- Include metric only when it exists in `03_Metric_Layer.md`.
- Connect to the target JD, not every project detail.

## 자기소개서

```text
공고 속 과제:
연결 경험:
내 행동:
결과:
입사 후 기여:
```

Avoid:

- "비전에 공감했습니다" alone.
- Generic growth motivation.
- Claims not backed by source career notes.

## STAR Interview Answer

```text
상황: 
과제:
행동:
결과:
배운 점:
```

Keep the spoken answer shorter than the written note. Use project IDs internally, but remove IDs from the final spoken answer unless the user wants a study note.

## 1-Minute Self Introduction

```text
현재 포지셔닝:
대표 경험 1:
대표 경험 2:
이 직무와의 연결:
```

Do not list every project. Pick one or two experiences that match the JD.

## Email Draft

```text
제목: [회사명] [직무명] 지원/문의드립니다

안녕하세요, [수신자]님.
[지원 목적/맥락]
[JD와 연결되는 핵심 경험 1-2문장]
[첨부/요청사항]
감사합니다.
```
