# plan-slides Skill

인터뷰 → 슬라이드 계획 제안 → 확인 → ppt.md 생성.

---

## 워크플로우

### Step 1 — 인터뷰 (4개 질문, 한 번에 묻기)

아래 4가지를 **한 메시지**에 담아 사용자에게 묻는다.

```
슬라이드 기획을 도와드릴게요. 아래 4가지를 알려주세요.

1. 발표 상황  — 어떤 자리인가요? (예: 투자 IR, 전략 보고, 제품 소개, 업무 보고, 세미나)
2. 청중       — 주요 청중이 누구인가요? (예: 투자자, 팀장, 고객사, 전사 임직원)
3. 핵심 메시지 — 발표 후 청중이 반드시 기억했으면 하는 것 1~3가지
4. 보유 내용  — 담고 싶은 데이터, 수치, 스토리가 있으면 자유롭게 적어주세요.
               (없으면 "없음"이라고 해도 됩니다)
```

사용자가 답하면 Step 2로 진행한다.

---

### Step 2 — 슬라이드 계획표 제안

인터뷰 답변을 바탕으로 슬라이드 구성을 제안한다.

**레이아웃 선택 기준** (make-slides/SKILL.md와 동일):
- 표지 → Cover
- 핵심 요약 2개 takeaway → Executive Summary
- 트렌드·옵션 3개 나열 → Three Trends
- AS-IS/TO-BE 비교 → Two Column Compare
- 문제 정의 / 현황 → Left-Title + Right-Two-Panel
- 특징 카드 4개 → Top-Bar + Card-Grid (4열)
- 핵심 수치 강조 → Metric Cards
- 단계별 프로세스 → Chevron Process
- 데이터 추이 → Chart — Line / Column
- 로드맵 → Timeline / Roadmap
- 목차 → Agenda
- 챕터 전환 → Section Divider
- 마무리 → Thank You

**계획표 형식:**

```
슬라이드 계획 (총 N매)
──────────────────────────────────────────────────────
 #  | 레이아웃               | 핵심 내용 요약
──────────────────────────────────────────────────────
 1  | Cover                 | 서비스명, 한 줄 설명
 2  | Executive Summary     | 핵심 takeaway 2개
 3  | Three Trends          | 주요 특징 3가지
...
──────────────────────────────────────────────────────
이대로 진행할까요? 순서·레이아웃 변경이 필요하면 말씀해 주세요.
```

사용자가 승인하면 Step 3으로, 수정 요청이 있으면 반영 후 다시 제안한다.

---

### Step 3 — ppt.md 생성

승인된 계획을 바탕으로 `ppt.md` 파일을 생성한다.

**출력 파일 위치:** 사용자가 지정한 경로, 또는 현재 작업 디렉토리의 `ppt.md`

**ppt.md 형식:**

```markdown
# [프레젠테이션 제목]

## 메타
- 발표 상황: [IR / 전략 보고 / 제품 소개 등]
- 청중: [대상]
- 핵심 메시지: [1~3가지]

---

## 슬라이드 1: [제목]
- 레이아웃: Cover
- 메인 타이틀: [제목]
- 서브 타이틀: [부제]
- 발표자: [이름]
- 날짜: [날짜]

---

## 슬라이드 2: [제목]
- 레이아웃: [레이아웃명]
- 핵심 메시지: [1줄]
- 내용:
  - [항목 1]
  - [항목 2]
  - [항목 3]

---
```

**작성 원칙:**
- 각 슬라이드마다 `레이아웃:` 명시 (make-slides가 이를 기반으로 레이아웃 선택)
- `핵심 메시지:` 1줄 — 슬라이드 한 장의 결론
- `내용:` bullet — 실제 슬라이드에 들어갈 데이터/문장
- 수치나 데이터가 있으면 최대한 구체적으로 기입
- 데이터가 없는 슬라이드는 `[placeholder]` 표시

---

### Step 4 — 완료 안내

```
ppt.md를 저장했습니다.

다음 단계: /make-slides 스킬을 실행하면 이 파일을 HTML 프레젠테이션으로 변환할 수 있습니다.
수정이 필요하면 ppt.md를 직접 편집한 뒤 /make-slides를 다시 실행하세요.
```
